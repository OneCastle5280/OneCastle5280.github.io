集合

集合与数组之间的区别

  1. 数组的长度是固定的，集合的长度是可变的
  2. 数组存储的是同一类型的元素，而集合存储的都是对象，并且对象的类型可以不一致

集合框架

collection接口

定义的是所有单列集合中共性的方法

所有 单列集合都可以使用的共性的方法

collection的常用的方法

List接口

特点
  1. 有序的集合（存储和取出元素顺序相同）
  2. 允许存储重复的元素
  3. 有索引，可以使用普通的for循环遍历
包括

Vector集合

ArrayList集合 ：底层是数组实现的，查询块、增删慢

LinkedList集合：底层是链表实现的，查询慢，增删快

Set接口

特点
  1. 无序的集合（存储和取出的顺序不一定相同）
  2. 不允许存储重复元素
  3. 没有索引（不能使用普通的for循环遍历）
包括

TreeSet 集合：底层是二叉树实现的，一般用于排序

HashSet集合：底层是哈希表+（红黑树）实现的，特点：无索引、不可存储重复元素、存取无序

​ LinkedHashSet集合：底层是哈希表+链表实现的，无索引、不可以存储重复元素、可以保证存取顺序

迭代器（iterator）

通用的遍历集合中元素的接口

即：主要用于迭代访问（即遍历）collection中的元素

iterator对象就称为迭代器

常用方法

hasNext()

如果有元素可以迭代，就返回true.

next()

返回迭代的下一个元素

实际使用

iterator是一个接口，我们无法直接使用，需要使用实现类来实现。

而collection接口中有一个方法，叫做iterator()，返回的就是基于该集合的迭代器

实现原理

底层有一个指针，指针初始指向 -1 索引

集合在定义的时候如果不指定类型

默认创建的就是Object类型，这个集合就变成了可以存储任何数据类型
